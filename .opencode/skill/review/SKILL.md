# Review

Use this when reviewing a diff or planning a final check.

Focus on:
- Bugs and regressions
- Unsafe filesystem behavior
- Missing clipboard validation
- Missing manual verification coverage
- Install or packaging risks

Output style:
- Findings first
- Highest severity first
- Cite file and line numbers when possible
