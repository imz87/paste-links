# Stories

This directory stores discussion-first product, architecture, refactor, and UX stories that are not yet implementation tasks.

Use stories for:

- rough ideas
- product or user problems
- architecture discussions
- discovery notes
- design direction
- work that may later become one or more tasks

Stories are not implementation specs. When a story is clarified and approved for implementation, the architect should extract one or more task files into `.docs/tasks/backlog/`.

Suggested story format:

    # Story

    Brief description of the problem or goal.

    # Context

    Why this matters.

    # Current Pain

    What feels wrong today.

    # Desired Direction

    What good looks like.

    # Notes / Decisions

    Important discussion outcomes.

    # Generated Tasks

    - `.docs/tasks/backlog/NNN-example-task.md`
